# MusicLesson API

REST API для демонстрационного экзамена «МузыкаУрок»: аутентификация, профили, каталог преподавателей, заявки на уроки и справочник инструментов.

Стек: Express, SQLite, Sequelize, JWT.

## Установка и запуск

```bash
cd backend
npm install
npm run migrate
npm run seed
npm start
```

Сервер: **http://localhost:3010** (или `PORT` из `.env`).

## Эндпоинты

Заголовок для защищённых маршрутов: `Authorization: Bearer <accessToken>`.

### Auth
- `POST /auth/register` — `{ email, password, name, role? }`
- `POST /auth/login` — `{ email, password }`
- `GET /auth/refresh` — refresh-токен в query, body или Authorization
- `POST /auth/refresh` — `{ refreshToken }`
- `POST /auth/logout` — выход

### Профиль
- `GET /profile/me` — текущий пользователь
- `PUT /profile/me` — обновление профиля
- `DELETE /profile/:id` — удаление аккаунта

### Преподаватели
- `GET /teachers?instrumentId=&studentLevels=&sortBy=rating&sortOrder=desc&limit=20&offset=0`
- `GET /teachers/:id` — профиль преподавателя

### Заявки на уроки
- `GET /lessons?search=` — список заявок (ученик/преподаватель)
- `POST /lessons` — `{ teacherId, date, time, comment }` (ученик)
- `PATCH /lessons/:id` — `{ status: "cancelled" | "accepted" | "rejected" }`

### Инструменты
- `GET /instruments?search=` — список инструментов
- `POST /instruments` — `{ name }` (преподаватель)
- `POST /instrument/selected` — `{ instrumentIds: [1, 2] }` (преподаватель)
- `DELETE /instrument/selected` — `{ instrumentIds: [1] }` (преподаватель)

## Уровни учеников

Значения для поля `studentLevels`: `beginner`, `advanced`, `professional`, `online`.

## Статусы заявок

`pending`, `accepted`, `rejected`, `cancelled`

Описание предметной области
Цель проекта: создать адаптивное веб-приложение для поиска преподавателей музыки и записи на индивидуальные уроки, с регистрацией пользователей, разграничением прав доступа и интерфейсами под разные роли: ученик, преподаватель, гость.
Вам необходимо сформировать архитектуру приложения, обеспечить защищённую аутентификацию и безопасное хранение информации.
Продумать интеграцию всех пользовательских сценариев: регистрация, авторизация, просмотр и фильтрация каталога, отправка и управление заявками на уроки.

Роли пользователей в системе
guest (гость) — может смотреть главную, каталог, детали профиля преподавателя, переходить на регистрацию/вход.
student (ученик) — guest + создавать/отменять свои заявки на уроки, смотреть «Мои уроки», редактировать свою анкету. Пункты меню и страницы «Авторизация» и «Регистрация» недоступны.
teacher (преподаватель) — может создать/редактировать свою анкету, просматривать заявки к себе и менять их статус (принять/отклонить), выбирать и добавлять инструменты. Пункты меню и страницы «Авторизация» и «Регистрация» недоступны.

Общие требования ко всем страницам
Все действия сопровождаются toast-уведомлениями и интуитивными анимациями.
На всех страницах до загрузки данных отображается Loader или Skeleton.
Все страницы должны содержать <h1> с названием страницы (например, «Каталог преподавателей»).
Все поля формы должны иметь label и placeholder (если применимо).
Обеспечить визуальное единообразие: использовать базовый стиль для всех элементов, соблюдать единую цветовую палитру, отступы, типографику.
Валидация: обязательные поля при пустом значении показывают текст ошибки под полем и поле подсвечивается (красная рамка).
Адаптивность: страницы корректно отображаются на двух точках ширины: 1920px и 370px. Меню и элементы страницы должны перестраиваться в списки. Контент не должен выходить за рамки экрана. Не должно быть горизонтальной полосы прокрутки.
Логотип, соответствующий теме приложения, в формате svg (должен корректно отображаться на всех страницах).
В консоли не должно быть ошибок (error).
Структура приложения соответствует архитектуре FSD. Используются лучшие практики проектирования и разработки кода — SOLID, DRY, KISS.

Основные страницы приложения

Страница регистрации (RegisterPage, /register)

Доступ: только неавторизованный пользователь.
Структура формы: email, имя, пароль, подтверждение пароля, роль; расположение полей: вертикальное.
Валидация (нельзя использовать валидацию html):
email — проверка на валидность, обязательное;
имя — от 5 до 15 символов, обязательное;
пароль — от 6 до 10 символов, обязательное;
роль — поле (select) с выбором роли — «ученик» или «преподаватель»; обязательное.
Действия:
после отправки формы, при некорректных значениях происходит отображение ошибок под полями или toast-уведомления; некорректные поля подсвечиваются красным;
при успешной регистрации осуществляется автоматический переход в заполнение профиля (/profile).
 


Страница входа (LoginPage, путь /login)

Доступ: только неавторизованный пользователь.
Структура формы:
Форма: email, пароль;
Расположение полей: вертикальное.
Валидация (нельзя использовать валидацию html):
email — проверка на валидность, обязательное;
пароль — от 6 до 10 символов, обязательное.
Действия:
после отправки формы при некорректных значениях происходит отображение ошибки «Неверный логин или пароль» или toast-уведомление.




Главная страница (HomePage, путь / или /teachers).

Доступ: все пользователи.
Структура страницы:
фильтры по инструменту (поле типа select со списком инструментов), уровню учеников (поле типа checkbox, значения — Начинающий, Продвинутый, Профессиональный, Онлайн);
элементы для сортировки по рейтингу (по возрастанию и убыванию);
список/сетка карточек преподавателей (до 20 элементов на странице). Карточка должна содержать: фото, имя преподавателя, инструмент, стоимость урока и рейтинг (в виде числа).
Действия:
при нажатии на карточку преподавателя происходит переход в его профиль (см. п. 2);
при изменении фильтров отображаются только подходящие карточки.
 


Просмотр профиля преподавателя (TeacherProfile, путь /teachers/:id)

Доступ: все пользователи.
Структура страницы:
подробная информация: ФИО, фото, о преподавателе, инструменты, стоимость урока, рейтинг, контакты;
кнопка «Записаться на урок» (см. «Отправка заявки на урок»); кнопка отображается только для роли «ученик». Для роли «преподаватель» кнопка не отображается.


Отправка заявки на урок (LessonForm, как часть TeacherDetailsPage либо модальное окно).

Доступ: только пользователи с ролью «ученик».
Структура страницы:
форма с выбором даты/времени (date и time поля), ввод пожеланий к уроку;
все поля формы обязательны для заполнения;
кнопка «Отправить заявку»;
индикация статуса (ожидание / успешно отправлено).
Действия:
если форма находится на странице просмотра профиля преподавателя — то она скрыта до нажатия на кнопку «Записаться на урок»;
в случае успешной отправки заявки отображается toast-уведомление с текстом «Ваша заявка успешно отправлена {name} на рассмотрение!».


Личный кабинет пользователя (ProfilePage, путь /profile)

Доступ: только авторизованные пользователи.
Структура страницы:
ссылка на страницу «Мои уроки»;
для преподавателя — ссылка на страницу с добавлением инструментов;
форма с возможностью изменить информацию пользователя: имя, email, пароль; для преподавателя — дополнительные поля «О преподавателе», фото (ссылка или файл), стоимость урока, рейтинг, контакты, уровень учеников (Начинающий, Продвинутый, Профессиональный, Онлайн);
кнопка «Удалить аккаунт», ярко красного цвета.
Действия:
изменение данных с возможностью их сохранить. Пользователю должна предоставляться обратная связь: успешно ли произошло изменение или список ошибок;
при нажатии на кнопку «Удалить аккаунт» должно выводиться окно подтверждения перед удалением аккаунта.


Страница выбора инструментов (Instruments, путь /instruments)

Доступ: только авторизованные пользователи с ролью «преподаватель».
Структура страницы:
список всех добавленных инструментов и checkbox для выбора;
форма поиска инструмента по названию;
форма для добавления нового инструмента (название).
Действия:
при установке checkbox происходит добавление инструмента к преподавателю; если checkbox снять — инструмент убирается;
при добавлении нового инструмента необходимо провести проверку на существование такого инструмента; если такой инструмент есть — вывести ошибку, иначе — добавить.


Список моих уроков (MyLessonsPage, путь /lessons)

Доступ: только авторизованные пользователи.
Структура страницы для роли «Ученик»:
таблица из трёх колонок: преподаватель (ссылка на профиль), дата и время, статус («ожидание», «принято», «отклонено»);
поле статус выполнено в виде бейджа с цветовой палитрой: ожидание — оранжевый цвет, принято — зелёный, отклонено — красный;
реализована возможность отменить заявку в виде кнопки «Отменить» рядом со статусом. Такая возможность доступна, только если заявка находится в статусе «ожидание»;
реализована форма поиска по имени преподавателя;
если заявок нет, отображается текст «Нет уроков».
Структура страницы для роли «Преподаватель»:
таблица из трёх колонок: ученик, дата и время, статус («ожидание», «принято», «отклонено»);
поле статус выполнено в виде бейджа с цветовой палитрой: ожидание — оранжевый цвет, принято — зелёный, отклонено — красный;
реализована возможность отклонить или принять заявку в виде кнопок «Отказать» и «Принять» рядом со статусом. Такая возможность доступна, только если заявка находится в статусе «ожидание»;
реализована форма поиска по имени ученика;
если заявок нет, отображается текст «Нет уроков».
